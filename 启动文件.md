# 同养缘项目 - 本地启动指南

本地开发调试时，请按以下顺序启动所有服务。

---

## 你的实际环境

> **重要**：你的电脑上 Docker Desktop 未运行。以下服务以替代方式提供：
> - **MySQL** → Windows 原生安装（PID 8024，端口 3306）
> - **Redis** → WSL Ubuntu 内运行（端口 6379）
> - Docker 层面的 `docker compose up` **不可用**，请忽略相关步骤
> - Redis **无密码**，已在 `application.properties` 中配置为空

---


```powershell
# 确认端口映射（确认 Windows 能访问）
netstat -ano | findstr "6379"
```

Redis 连接信息（来自 `application.properties`）：

| 地址 | `localhost:6379` |

## 手机端测试配置

修改 `NetworkConfig.java` 里的 `ENV_MODE`：

| 值 | 含义 |
|----|------|
| `0` | 生产：远程 `106.120.183.117` |
| `1` | **Android Studio 模拟器**（本机后端） |
| `2` | 真机 + 本机后端（填 `LOCAL_LAN_BASE_URL` 为电脑局域网 IP） |

### 模拟器（必做：端口转发）

在部分 Windows / Hyper-V 环境下，`10.0.2.2` 连不上本机后端。项目已改为使用 **`127.0.0.1:8080` + `adb reverse`**。

1. 先启动模拟器，再启动本机后端（8080）。
2. 双击项目根目录 **`adb-reverse-8080.bat`**，或在终端执行：

```powershell
"%LOCALAPPDATA%\Android\Sdk\platform-tools\adb.exe" reverse tcp:8080 tcp:8080
```

3. 确认 `ENV_MODE = 1`，然后 **Run App**。每次新开模拟器或 `adb` 重连后，建议再执行一次 `adb reverse`。

### 真机测试

**方式一：修改 NetworkConfig.java**

```java
private static final int ENV_MODE = 2;
// 并把 LOCAL_LAN_BASE_URL 改成你电脑的 IPv4，例如 http://192.168.1.5:8080/api
```

查询本机 IP：

```powershell
ipconfig
```

找到类似 `192.168.x.x` 的地址，确保手机和电脑在同一局域网下。

**方式二：使用内网穿透（推荐，方便异地测试）**

使用 ngrok 将本地 8080 端口映射为公网地址：

```powershell
# 安装 ngrok（如果没有）
# 注册账号：https://ngrok.com

ngrok http 8080

# 会生成一个公网地址，例如 https://xxxx.ngrok.io
# 修改 NetworkConfig.java 的 PRODUCTION_BASE_URL 为该地址
```

将以下内容保存为 `start-dev.ps1`，放在项目根目录（D:\AllProject\AndroidStudioProjects\TYY\），双击即可启动：

```powershell
Write-Host "`n=== 步骤4：启动咨询师端页面 ===" -ForegroundColor Cyan

Write-Host "咨询师端启动中..." -ForegroundColor Yellow

Write-Host "  全部服务已启动！" -ForegroundColor Green

Write-Host "  前端 Web：http://localhost:8002" -ForegroundColor Green
```

## 常见问题

### Q1: 后端启动成功但 API 请求一直转圈/超时

PowerShell 的 `Invoke-WebRequest` 与 Tomcat 有兼容问题，**一律用 `curl.exe`**：

```powershell
# ❌ 错误方式（会卡死）
Invoke-WebRequest -Uri "http://localhost:8080/api/auth/sendCode" -Method POST

# ✅ 正确方式
curl.exe -s -X POST "http://localhost:8080/api/auth/sendCode" -d "phone=13800138000"
```

### Q2: Redis 连接超时

WSL Redis 无需密码。如果之前配置过密码，先清除：

```powershell
wsl -d Ubuntu redis-cli CONFIG SET requirepass ""
```

### Q3: Maven 启动很慢

```powershell
# 先确保 Maven 能联网，然后手动编译
cd "D:\AllProject\AndroidStudioProjects\TYY\TongYangYuan-Server"
mvn clean install -DskipTests
mvn spring-boot:run
```

### Q4: 前端 `npm install` 失败

```powershell
# 清除缓存后重试
cd "D:\AllProject\AndroidStudioProjects\TYY\TongYangYuan-Server\web"
Remove-Item -Recurse -Force node_modules
npm cache clean --force
npm install
```

### Q5: MySQL 连接被拒绝

确保 Windows MySQL 服务在运行：

```powershell
# 检查 MySQL 进程
Get-Process -Id 8024 -ErrorAction SilentlyContinue | Select-Object ProcessName

# 如果没运行，通过服务管理器启动 MySQL
services.msc  # 手动启动 MySQL 服务
```

### Q6: 每次都要重新启动所有服务太麻烦

MySQL 和 Redis 只要电脑不重启就一直运行，后续只需启动后端和前端即可。
