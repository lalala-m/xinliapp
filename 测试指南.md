## 二、功能测试

### 2.1 订单系统测试

#### 创建订单

```bash
# 创建会员订单
curl -X POST http://106.120.183.117:8080/api/orders \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "orderType": "ORDER_TYPE_MEMBER",
    "packageName": "童康源年度会员",
    "originalPrice": 365.00,
    "discountAmount": 50.00,
    "actualPrice": 315.00,
    "vipValidDays": 365
  }'
```

#### 模拟支付（测试用）

```bash
# 假设返回的订单号是 TYY20260323xxxxxx
curl -X POST http://106.120.183.117:8080/api/orders/TYY20260323xxxxxx/simulate-pay \
  -H "Content-Type: application/json" \
  -d '{"paymentMethod": "ALIPAY"}'
```

#### 查看用户订单

```bash
curl http://106.120.183.117:8080/api/orders/user \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 三、API 接口汇总

### 3.1 订单相关

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/orders` | 创建订单 |
| GET | `/orders/{orderNo}` | 获取订单详情 |
| GET | `/orders/user` | 获取用户订单列表 |
| POST | `/orders/{orderNo}/simulate-pay` | 模拟支付（测试用）|
| POST | `/orders/{orderNo}/cancel` | 取消订单 |
| POST | `/orders/callback` | 支付回调 |

### 3.2 通话相关

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/call/config` | 获取通话配置 |
| POST | `/call/session` | 创建通话会话 |
| GET | `/call/session/{sessionId}` | 获取会话状态 |
| GET | `/call/pending?userId=` | 获取待接听来电 |
| POST | `/call/{sessionId}/accept` | 接听 |
| POST | `/call/{sessionId}/reject` | 拒绝 |
| POST | `/call/{sessionId}/end` | 结束 |

### 3.3 LiveKit Token

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/livekit/token?room=&identity=` | 获取 Token |
| GET | `/livekit/config` | 获取配置 |

## 四、测试模式说明

### 4.1 订单系统

- **真实支付**：需要配置支付宝/微信商户号
- **测试支付**：使用 `/simulate-pay` 接口模拟支付

### 4.2 音视频通话

- **真实通话**：需要部署 LiveKit 服务器
- **模拟通话**：当前配置为模拟模式，UI 流程正常但不实际连接媒体服务器

## 五、常见问题

### 5.1 订单创建失败

- 检查数据库是否创建了 `orders` 表
- 检查用户是否已登录（有 Token）

### 5.2 通话无法连接

- 检查 LiveKit 是否已部署
- 检查防火墙是否开放了 LiveKit 端口（默认 7880, 7881）

### 5.3 无法收到来电通知

- 检查 OpenIM 是否已部署
- 检查 APP 是否在线
